# Code of Conduct

Shlink is an open source, free hobby project that is provided as is. Take it as a gift, you can either accept it or move on, but never consider yourself entitled to anything.

The project does not accept external contributions, but you are free to open feature requests, bug reports, security advisories or discussions.

## Project roadmap

Due to Shlink's hobby project nature, there will **never** be anything remotely similar to a roadmap. I work on it in my own free time, when I have the energy and nothing better to do.

Avoid asking for when will something be done, if there has been any progress on a particular task or complain about how long an issue has been opened. That kind of question will be simply ignored.

If you are interested in knowing on a particular issue's progress, subscribe to it and you will be notified when something changes.

## Strict No LLM / No AI Policy

The usage of LLMs/AI to repport issues, discussions or security advisories is **strictly prohibited**, and will lead to closing and discarding those reports without further explanation.

In extreme cases, it might lead to the permanent ban of the reporter's user account.

## Constructive behavior only

Always use safe and constructive behaviors, that contribute to a positive environment.

## Language

Always use English when interacting with this project's repositories, unless you don't know the language. In that case, use the one you are more confortable with.
